1 基本概念
	1.1 repository(仓库)
	一个项目对应一个仓库
	1.2 star(收藏)
	方便下次查看项目
	1.3 fork(复制克隆项目)
	注意复制的项目是独立存在于自己git中(会有fork from xxx/xxx仓库),如果在复制的仓库中更改,是不会改变原有的项目的
	1.4 pull request(发起请求)
	发起合并请求，基于fork.如果需要更新原有项目,可以发起pull request,在原有项目的管理员同意之后,就可以进行同步
	1.5 watch(关注)
	第一时间得到项目信息
	1.6 issue(事物卡片)
	事务卡片，发现代码bug，但目前没有成型代码，需要讨论时可用.或者使用开源项目出现问题时使用
	1.7 github主页
	1.8 仓库主页
	1.9 个人主页

2 注册github账号
	github主页 http://github.com
	2.1注意事项
	2.1 github在国外服务器,所有国内访问慢或者无法访问,需翻墙(shadowsocks)
	2.2 私有仓库只能自己或者制定的朋友才有权限操作(私有仓库是收费的)
	2.3 新注册的用户必须要验证邮箱后才能使用

3 仓库管理
	新建文件(create new file) 文件标题,文件内容,文件提交说明(每次提交的目的,方便自己/其他开发者查看)
	3.1 编辑文件(edit file)
	可以在网页中修改文件,每次编辑算一次commit
	3.2 删除文件(delete file)
	删除的文件详细信息可以在Commits中查看，Commits可以查看每次修改的相关信息
	3.3 上传文件(upload file)
	3.4 搜索仓库文件(find file)
	3.5 下载/检出项目(clone or download)

4 github issue
	发现bug,需要讨论时候使用.或者使用开源项目出现问题时使用.提交使用问题或者建议或者想法

5 发起请求 pull request
	步骤
	5.1 fork项目
	5.2 修改自己仓库的项目代码
	5.3 新建pull request
	5.4 等待作者操作（合并）
